# Twitch チャンネルポイント演出管理システム 要件定義書

## 1. 機能要件

### 1.1. チャンネルポイント報酬の管理
* **報酬の作成・編集・削除**: チャンネルポイント報酬のメタデータをローカルで作成および管理する機能。
* **各種設定**: 以下の項目を報酬ごとに設定可能とする。
  * 必要ポイント数
  * クールタイム（再使用制限時間）
  * ロール制限（特定の視聴者のみ有効化など）
* **Twitch チャンネルポイント自動取得・同期**: Twitch API からチャンネル上に存在する全カスタム報酬を自動取得し、アプリ内の報酬一覧とシームレスに同期・マージする機能。IDなどのコピペを一切不要とし、一覧から選ぶだけで演出の紐付けができる。未設定のものは薄灰色（コントラスト配慮済）、設定済みのものは緑色で分かりやすく色分け表示する。
* **複数演出（エフェクト）の設定**: 1つの報酬に対して複数の演出（画像や音等）を紐付け、順番再生（sequential）またはランダム再生（random）する機能。
* **マルチ演出コントロールと安全ガード**: 報酬編集画面上で演出効果を任意に新規追加・削除可能とし、編集中の変更を一時バッファリングする機能。ただし、演出が0個になってしまうのを防ぐため「最後の1個のエフェクトは削除させない（削除ボタンの自動無効化・警告保護）」という安全ガードを設ける。
* **「カスタムHTML演出として実行」モード**: 演出ごとに HTML ファイルを紐付け、チャンネルポイント使用時にそれらをカスタムHTML演出として実行するモード。HTML はOBSのiframe経由でオーバーレイ表示する。

### 1.2. 演出の表示
* **多様なメディアの表示**: 画像、GIF、動画、音声、テキストを表示可能とする。
* **制御機能**: 画面の任意の場所へのピクセル単位での表示位置指定（自動見切れ補正付き）、表示サイズのスケーリング（1～100%）、表示時間、アニメーションの制御機能。
* **キューイング**: 複数の演出リクエストが同時に発生した場合、順番に遅延なく処理する機能。
* **音量の個別調整**: 報酬ごと、および演出（アセット）ごとに音量レベルを設定可能とする機能。
* **演出の強制停止（パニックボタン）**: 実行中の演出およびキューイングされている演出を、管理者が即座にすべて停止・消去する機能。
* **透過動画のサポート**: アルファチャンネル対応の動画（WebM等）の再生を可能とし、背景が透き通ったリッチな演出を実現する機能。
* **カスタムHTML演出**: カスタムHTML演出モード時は、HTML ファイルを OBS ブラウザソースの iframe に全画面表示する（WebSocket `show_custom_html` メッセージ経由）。

### 1.3. 管理機能
* **リアルタイムダッシュボード**: キューの蓄積状況や、本日の報酬使用回数等をリアルタイムで確認できる機能。
* **使用履歴・統計**: 報酬全体の利用回数（再生ランキング）に加え、ユーザごとの個別の報酬利用回数を横断的に閲覧できる機能。
* **キュー管理**: 順番待ちしている演出キューの全クリア、または特定の演出の個別削除機能。
* **プレビュー機能**: 設定した演出が実際にどのように表示されるかを、Twitchを介さずにアプリからテスト再生できる機能。
* **ランキング用カスタム表示**: `ranking/` フォルダに HTML ファイルを配置することで、OBS ブラウザソースからファイル名を指定してランキング等の独自表示が可能。`/ranking/<ファイル名>` エンドポイント経由でアクセスし、静的ファイルとして配信される（配信時に HTML サニタイザーが自動適用され、危険なコードが排除される）。


### 1.4. 連携
* **Twitch OAuth認証と自動ライフサイクル管理**:
  * 配信者アカウントを用いたセキュアなログインとトークンの取得。
  * **プロアクティブ自動リフレッシュ**: トークンの有効期限（通常4時間）による機能停止を防ぐため、アプリ起動時およびアプリ稼働中の一定時間ごと（2時間おき）にバックグラウンドで自動的にアクセストークンを最新状態に更新する機能。
  * **エラーのインテリジェントな判別**: 自動リフレッシュ失敗時、一時的なエラー（ネットワーク不通など）と恒久的なエラー（クライアント情報の誤りやトークン無効化）を自動判別する機能。
    * 一時的エラーの場合は、配信を邪魔しないよう警告を出さず、バックグラウンドで自動リトライを行う（無限リトライを避けるため、最大5回までに制限し、上限に達した場合はリフレッシュ処理を安全に停止する）。
    * 恒久的エラーの場合は、タイマーを停止し、UIに「再連携」を促す警告を表示する。
* **Twitch EventSub（チャンネルポイント監視）**: チャンネルポイント報酬が引き換えられたイベントをリアルタイムで検知する機能。
* **OBSブラウザソース**: 作成した演出をOBS上でオーバーレイ表示するため、ブラウザソースとして組み込める仕組み。

### 1.5. セキュリティ・安全機能
* **パストラバーサル防止**: `/ranking/<ファイル名>` エンドポイントでは、`ranking/` フォルダ外へのディレクトリトラバーサルアクセスを禁止し、HTTP 403 Forbidden を返す。
* **静的配信の安全性**: `ranking/` フォルダ内のファイルは、サーバーサイドスクリプト（PHPやCGI等）を一切実行せず、静的HTMLおよび関連アセット（CSS, 画像など）のみを安全に配信する（JavaScriptはサニタイザーにより自動的に完全排除される）。
* **ゼロトラストHTML/CSSサニタイザーによる防御**: カスタムHTML演出、およびランキング表示（/ranking/）のHTML配信時に、C++ネイティブのサニタイザーにより配信直前に以下のクレンジングが強制的に適用され、XSSや外部接続、悪意あるコードの埋め込みが100%防止される。
  * **JavaScriptの完全排除**: `<script>`ブロックの中身ごと全消去、すべてのイベント属性（`on*`）の完全消去、`javascript:` スキームの無効化。
  * **HTMLタグのホワイトリスト制限**: `div`, `span`, `p`, `br`, `table`, `tr`, `td`, `img`, `style` のみ許可。それ以外のタグ（`iframe`, `object`, `embed`, `svg`, `canvas`, `link`, `form`, `input`, `button` 等）はタグごと消去。
  * **ローカル画像限定ポリシー**: `img` タグの `src` 属性は、ローカル画像ファイル（PNG, JPG, JPEG, GIF）のみ許可し、外部URL（http, https）やSVG画像、インライン埋め込み（`data:` スキーム）は除去。
  * **CSSの外部接続の遮断**: `style` タグや style 属性内の `@import` や `url(...)` の記述を自動消去し、外部フォントや画像、トラッキング接続を遮断。

## 2. 非機能要件

### 2.1. 技術スタック
* **デスクトップアプリ**: Qt 6.x + C++20
* **ローカルデータ保存**: SQLite または JSON

### 2.2. 制約・性能要件
* **ローカル完結動作**: ルーターのポート開放や、外部プロキシサーバーなどの契約を不要とし、デスクトップアプリ単体で動作を完結させる。
* **低リソース使用**: 配信者が同一PCでPCゲームやOBSなどを同時に実行する過酷な動作環境を考慮し、バックグラウンドでの動作時を含めて極めて低いCPU・メモリ使用率を維持する。
* **リアルタイム処理**: 視聴者がTwitch上でチャンネルポイントを使用してから、OBS上に演出が表示されるまでの遅延（レイテンシ）を最小化する。

### 2.3. セキュリティ
* **OAuth認証情報の安全な保存**: 取得したアクセストークンやリフレッシュトークン等の認証情報をセキュアに保管する。
* **コンテンツフィルタリング方針**: 演出として登録されるアセット（音声や画像など）に対して、アプリ側での自動フィルタリング等は行わない（配信者の自己責任とする）。

### 2.4. 拡張性
* **アニメーション追加の容易性**: 将来的な演出エフェクトやアニメーションの追加が容易なコード設計。
* **将来的な機能追加に対応可能な構造**: 今後の要件変更（例：他プラットフォームとの連携等）に備えた疎結合な設計。

### 2.5. ユーザビリティ・アクセシビリティ
* **高コントラスト・ユニバーサルダークデザイン**: 暗い背景上でも、すべてのテキスト（QCheckBoxラベルやQListWidgetの未設定アイテム等）がクッキリと読めるよう十分なコントラスト比（#E1E1E6 明灰色〜#FFFFFF 白色など）を保証する設計。
* **一貫したダイアログスタイル**: アプリ内メッセージや警告表示用ダイアログ（QMessageBox等）の背景およびボタンが白浮きせず、全体テーマと一体化した高品質なダーク調でレンダリングされること。
